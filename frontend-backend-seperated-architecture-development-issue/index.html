前后端分离架构下的常见低级错误
代码层面
数据属性命名混乱
id前面带实体名
[{
programName
programId
}]
数据后面不应该跟后缀
后缀往往是前端组件的名字，在数据上不应该有
sectionList
sectionDetail
乱用状态码
单复数混乱
函数命名混乱
滥用匿名函数，不起名字
封装混乱
一个Section的Validator封装一个类
滥用闭包，修改全局变量
实体命名混乱
SectionSubmit-其实是section level的answerSheet
实体关系混乱
谁应该有一个根级URL的问题
一个实体可以跨根级URL而存在，且用他的时候不需要知道他的父节点，他就应该有一个根级URL
正面举例
Homework
Logic
反面举例
BasicQuiz
OrderItem
逻辑分层混乱
一个逻辑应该在Backend还是BFF还是Frontend
数据计算，应该在Backend
BFF负责的是
调度
访问后台API
过滤／重组
掩盖不该被前台访问的数据
重组数据结构，方便前台操作
暂存
权限
界面逻辑在Frontend
乱加缓存
不该存起来的存起来
考卷定义
该存起来的没存起来
BFF耦合
teacher与web混为一个（这还是一个典型的命名错误）
技术栈层面
每个层级之间的接口处缺少打印
redux的state没打印
bff接受的请求没打印，bff返回的响应没打印
bff向Backend发出的请求没打印，bff从backend接受的响应没打印
backend接受的请求没打印，backend返回的响应没打印
backend的sql没打印
开发模式下的实时更新
Frontend不是实时更新的
BFF和Backend的实时重启
API不支持版本
结果导致重构的时候，牵一发而动全身
